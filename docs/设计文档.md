# VoiceDraw 设计文档

> 七牛云 × XEngineer 暑期实训营 · 题目二「AI 语音绘图工具」
> 纯语音控制、不用鼠标键盘，完成「精确矢量绘图 + AI 创意生图」的语音创作工作台。

---

## 1. 产品概述与定位

VoiceDraw 是一款**纯语音控制**的绘图工具：用户全程不碰鼠标键盘，仅用口述指令完成创作。它把两类需求合到一块画布上：

- **效率模式（矢量工作台）**：确定性绘图——图形、颜色、变换、增删改，以及"一句话出图表"（流程图 / 组织架构图 / 思维导图 / 树状图 / 线框图）。本地规则驱动，响应 < 100ms、零成本、可离线。
- **创意模式（AI 画面）**：文生图 + 语音局部重绘 + 作品库，由云端大模型生成与编辑。

设计目标对齐题目三大评分维度：**指令理解的准确性与容错性、语音到绘图的响应延迟、复杂指令的拆解与执行能力**，并额外强化**端云协同的成本控制**与**产品完整度/创新性**。

吉祥物「小绘」（一只暖萌画家小熊）作为语音搭档贯穿全程，用待机 / 说话 / 绘画三段动画在合适时机给出陪伴式反馈。

## 3. 系统架构（端云协同）

```
[麦克风] → 阿里 DashScope 实时流式 ASR(paraformer-realtime-v2) → 实时字幕
                                  │
                          [指令路由 router]（去抖合并整句）
                                  │
        ┌──────────────┬──────────┴───────────┬───────────────────┐
        ▼              ▼                      ▼                   ▼
  本地规则引擎     LLM 语义理解            结构图引擎           创意（云端）
  (高频原子指令)   (复杂/自然语句兜底)   (LLM抽结构+dagre排版)  文生图/局部重绘
   <5ms 0成本      七牛 chat              流程/架构/脑图        七牛 gemini 图像
        │              │                      │              + 对话式确认(qwen-vl 看图)
        └──────────────┴───────┬──────────────┴───────────────────┘
                               ▼
              画布对象管理(zustand) + SVG 渲染 + 撤销/重做
                               ▼
                语音反馈(TTS) + 小熊状态动画 + 成本仪表盘
```

- **前端**：Vite + React + TypeScript + zustand；画布用 **SVG 场景图**（矢量与 AI 位图同层、易加滤镜、易导出 PNG/SVG）。
- **后端**：极简 Node/Express 代理，**持有密钥不暴露浏览器**、免 CORS，并内置**成本计量、生图缓存、429 限流自动重试**。
- **云端能力**：七牛云 AI 大模型推理（OpenAI 兼容，一把 key 覆盖 对话理解 / 文生图 / 局部重绘 / 视觉看图）；语音识别用阿里 DashScope 实时流式 ASR。
- **优雅降级**：缺 key 自动隐藏对应云端功能，本地绘图照常可用，绝不假装有效果。

## 5. 关键能力详解

### 5.1 指令理解的准确性与容错性
- **本地优先 + LLM 兜底**：高频原子指令走本地规则引擎（确定性、不会幻觉、可离线）；复杂/自然语句兜底给大模型解析为结构化操作。
- **语音分句合并（去抖）**：把"换气微停顿"被 ASR 拆碎的多段，合并成一条完整指令再执行，根治"一句话被拆成多次、各触发一次"。
- **意图理解而非照搬**：口语（含"嗯/呃/不对…"）经大模型理解为**最终真实意图**的干净提示词，而非原文照搬。
- **模糊匹配与防误触**："文本框"="输入框"；"在左边画圆"识别为绘制而非移动；编辑措辞不误触发"重新生成整图"。

### 5.2 语音到绘图的响应延迟
- 效率模式本地指令**全程本地、<100ms 即时渲染**。
- 流式 ASR **边说边出实时字幕**；停顿后自动执行。
- 创意模式生图为云端异步，配**加载动效 + 语音过渡 + 出图揭示动画**，并以"对话式确认"把昂贵请求压到最少。

### 5.3 复杂指令的拆解与执行
- **语音 → 逻辑 → 图表**：一句自然语言描述（"登录流程：输入账号密码，验证，对就进首页…"），系统自动解析出起点/操作/判断/终点与连线条件，并完成专业排版。这接近"自然语言编程式绘图"。

### 5.4 对话式确认（创新交互，解决限流 + 容错）
生图/改图这类**昂贵且受限流**的操作，在真正发请求前先确认：
1. 用户口述意图 → 系统理解；
2. **改图时用 qwen-vl 视觉模型真的看一眼当前图**，生成"确认话术 + 精炼编辑指令"（例："要把图中央碗里浸水的绿苹果改成红色吗？"）；
3. 用户说"对"才真正发请求；说"不对，应该是…"则带修正重新确认。

一招治三病：**降低调用次数（治限流）、确认即纠错（治乱断句/识别错）、看图后再发请求（更准）**。

## 6. 创新点

1. **双引擎混合画布**：同一画布上既能"用嘴搭积木"画精确线框/图表，又能直接"生成氛围画面"，两者可叠加语音操作——结构化与非结构化混合创作。
2. **语音 → 逻辑 → 图表**：把自然语言逻辑链自动变成排版好的流程图/架构图，接近自然语言编程的绘图体验。
3. **对话式 + 视觉确认**：发请求前先看图、先确认、可纠错，是对"一次生成不可控"的体验突破。
4. **角色化语音搭档**：暖萌小熊「小绘」三段动画在待机/说话/绘画时切换，把工具变成有陪伴感的创作伙伴。
5. **实时成本仪表盘**：把"端云协同成本控制"可视化为可信证据。

## 7. 端云协同成本控制

| 策略 | 做法 | 效果 |
|---|---|---|
| 本地优先 | 高频指令走规则引擎，0 次 LLM 调用 | 省最大头 |
| 小模型 + 短 prompt | NLU 用便宜模型 + JSON 模式，仅在规则未命中时调用 | 降单价 |
| 生图缓存 | 相同提示词命中缓存不重复生图 | 省重复 |
| **对话式确认门** | 只有用户确认才真正发生图/改图请求 | 大幅压低昂贵调用、天然限速 |
| 限流退避重试 | 撞 RPM(429) 自动退避重试 | 提稳定性 |
| 实时计量 | 后端统一统计调用数与估算成本，前端成本仪表盘展示 | 可观测 |

## 8. 技术栈

- 前端：Vite · React · TypeScript · zustand · SVG 渲染 · @dagrejs/dagre（图表布局）· Geist 字体 · Phosphor 图标
- 后端：Node · Express 代理（持密钥 / 计量 / 缓存 / 重试）
- 语音：阿里云 DashScope `paraformer-realtime-v2`（流式 ASR）· 浏览器 SpeechSynthesis（TTS）
- 大模型：七牛云 AI 大模型推理（OpenAI 兼容）— 对话理解、`gemini-2.5-flash-image`/`gemini-3.0-pro-image-preview`（文生图/局部重绘）、`qwen-vl-max`（视觉看图确认）

## 9. 未完成与取舍说明（诚实记录）

| 项 | 状态 | 原因 |
|---|---|---|
| 像素级自由局部涂抹 | 取舍为"指令式 + 视觉确认" | 纯语音无参考点的像素级涂抹不可靠、延迟易超标；基于体验红线主动转化 |
| 图表删节点后自动重排 | 就地编辑（不自动重排） | 需缓存图结构后重跑布局，列入后续优化 |
| 局部重绘连发受 RPM 限制 | 用"确认门 + 冷却 + 退避重试"缓解 | 免费档每分钟请求数上限；如需更高频可切更高档接口 |
| 麦克风环境 | 推荐独立 Chrome | 浏览器安全策略：需在独立页面授权麦克风（嵌入式预览不支持） |

## 10. 演示脚本（约 4 分钟）

1. **效率模式**：说"画一个红色的圆""在右边画一个蓝色矩形""大一点""旋转45度""撤销"——展示本地即时、准确、可在同一画布增删改。
2. **一句话出图表**：说"画一个公司组织架构图"→ 自动出树状图；再说"把技术部改成研发中心""把后端组删掉"——展示复杂指令拆解 + 就地编辑。
3. **创意模式**：说"切换到创意模式""画一幅雨夜东京霓虹小巷，赛博朋克风格"→ 弹出**对话式确认**（小绘看懂并复述）→ 说"对"→ 出图；再说"把天空改成暖橙色"→ 看图确认 → 局部重绘。
4. **收尾**：展示作品库、导出、成本仪表盘，并说明"未完成的自由涂抹是基于体验红线主动砍掉的"——体现产品决策逻辑。
