# 架构与技术选型决策日志（ADR）

> 给汇报/路演/答辩用：记录关键决策、**为什么这么选**、以及真实的实测与踩坑过程。
> 这些"踩坑→验证→修正"的过程，正是工程质量与决策能力的体现。

---

## ADR-1：本地规则引擎 + LLM 兜底的"混合解析"

**决策**：高频原子指令走**本地规则引擎**（确定性代码），复杂/自然语句才走 **LLM**。

**为什么**：题目把"响应延迟"和"成本控制"列为评分点。
- 本地：<100ms、零成本、确定性（不会幻觉）、断网可用、演示不翻车；
- LLM：适合语义理解（复杂语句、流程图逻辑拆解、线框图），但有 500ms~2s 延迟和 token 成本。

**关键洞察**：流程图**不能全走本地**——"理解逻辑"是语义任务（用 LLM 抽节点/边），"排版"是确定性算法（用 dagre 本地算）。即"本地管快而确定的事，LLM 管难而模糊的事"。

## ADR-2：图像/对话 API 选型 —— 七牛 Gemini 为主

**决策**：云端能力（NLU + 文生图 + 局部重绘）全部用**七牛 AI 大模型推理**（`https://api.qnaigc.com/v1`，OpenAI 兼容）。

**对比过的候选**：

| 维度 | 七牛→Gemini | 硅基流动→FLUX Kontext | 豆包 Seedream/SeedEdit | OpenAI gpt-image-1 |
|---|---|---|---|---|
| 局部重绘(语音友好/免蒙版) | ✅ Nano Banana 强项 | ✅ | ✅ | ⚠️ 需蒙版 |
| 接入复杂度 | ✅ 一套 key 全包 | 需单独 key | 另接火山平台 | 需国际网络 |
| 国内访问/计费 | ✅ 直连/人民币/有免费额度 | ✅ | ✅ | ❌ |
| 贴合本次实训营 | ✅✅ 赞助方自家平台 | 中 | 中 | 低 |

**决定性逻辑**：我们的局部重绘是**纯语音**的——用户没法用嘴画蒙版，所以必须选"免蒙版、自然语言指令编辑"的模型（Gemini Nano Banana / FLUX Kontext / SeedEdit），这直接排除了需要蒙版的 gpt-image-1。而七牛恰好代理了 Gemini 图像模型，**同时拿下"编辑天花板 + 贴合赞助方 + 一套 key 全包"**。备选保底：硅基流动 FLUX Kontext（同为 OpenAI 兼容，一行配置可切）。

## ADR-3：文生图/局部重绘必须走 `/chat/completions`（而非 `/images/generations`）

**这是踩坑得出的关键结论。**

- 一开始按直觉用 `POST /images/generations` + `gemini-2.5-flash-image`，HTTP 200 且返回了图片——**但看真图发现完全跟 prompt 无关**（"红苹果"出来一辆蓝色跑车，"柴犬"出来森林拱门）。**若只看状态码就会被骗。**
- 改用 `POST /chat/completions`（Gemini 原生就是多模态对话出图），图片在 `choices[0].message.images[0].image_url.url`（data URI）。这条路 prompt 才被正确执行。

**教训**：验证生成类 API，必须看**真实产物**，不能只看 HTTP 200 / 字段存在。

## ADR-4：生成默认用 Pro 模型 + modalities，编辑用 flash

**实测对比（同一 prompt "戴墨镜的柴犬，扁平插画，纯蓝底"）：**

| 配置 | 结果 | 贴合度 |
|---|---|---|
| flash，无 modalities | 写实的猫 | 差 |
| flash + modalities | 忍者猫+繁复背景 | 差 |
| **Pro(gemini-3.0) + modalities + 明确出图指令** | 扁平矢量风格的狗、居中 | **明显更好** |

- **结论**：纯生成 **flash 贴合度弱、Pro 明显更好**，故**生成默认用 Pro**（约 20s，配加载动画+语音反馈可接受）。
- **场景图实测优秀**：Pro 画"雨夜东京霓虹赛博朋克"得到高质量、强贴合的电影感街景——而创意模式真正的用法就是场景/氛围图，所以这个选择对路。
- **编辑（局部重绘）用 flash**：实测"绿苹果→红苹果"精准、又快（~10s），对输入图强约束使其无需 Pro。
- 精确多属性物体（配饰/品种/底色）受底层模型能力限制仍有偏差，已如实记录在设计文档"未完成原因"。

## ADR-5：后端代理（不让密钥进浏览器）

**决策**：用极简 Express 后端持有七牛 key，前端只与 `/api` 通信。

**为什么**：① 密钥安全（题目仓库 6/15 要公开）；② 免 CORS；③ 后端天然适合做**生图缓存 + 成本计量**，正好把"成本控制"做成可演示证据。`.env` 已 gitignore，并约定公开前 rotate key。

---

## ADR-6：SVG 场景图 + 原生滤镜

**决策**：画布用 **SVG**（而非 Canvas 位图）。

**为什么**：声明式、从状态对象直接渲染、易做选中/变换/导出、易叠加 AI 位图（`<image>`）、**风格滤镜直接用原生 `<filter>`**（水彩=湍流位移、霓虹=高斯发光、黑白=去饱和…）零成本实时，无需图像 API。

## 踩坑与排障记录（真实过程）

1. **环境变量污染**：`ANTHROPIC_SMALL_FAST_MODEL=deepseek-v4-flash`（用户早先接 DeepSeek 的遗留）导致 WebSearch/WebFetch 报错——删除用户级环境变量并重启修复。
2. **`/images/generations` 对 Gemini 无效**：见 ADR-3。
3. **`.env` 空字符串覆盖默认值**：`NLU_MODEL=` 留空会被读成空串、覆盖代码默认值，七牛报 "Model required"。改用 `process.env.X || 默认值` 兜底。
4. **RPM 限流（429）**：免费额度有每分钟请求数上限，密集调用会 429——更印证"本地优先+缓存+防抖"的成本策略必要性。
5. **图像额度欠费（403 access_denied）**：对话仍可用、图像被停，充值后恢复。
6. **端口残留**：Windows 下 dev 进程被停后子进程仍占 5173/8787，需 taskkill 清理；最终用"preview 托管前端、Bash 跑后端"的分工避免冲突。
7. **大图截图超时**：2.9MB data-URI 背景图让预览截图器超时，改用 `eval` 读取 DOM 状态验证功能正常（功能无问题，是工具限制）。
